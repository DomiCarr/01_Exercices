Définitions

Sélecteurs sémantiques HTML

header

Entête de la page: Logo, titre du site ou d'une section

nav

Section de navigation, généralement utilisée pour les menus principaux ou secondaires

main

Contenu principal et unique de la page. Ne doit pas inclure les éléments répétitif tels que header, nav, footer, ...

article

sert à définir du contenu indépendant tel que blog, actualité, commentaire, ...

section

Sert à regrouper du contenu thématiquement cohérent. Doit avoir un titre , généralement h2

aside

Contenu secondaire comme barre latérale, publicités, liens connexes ...

footer

Pied de page : droits auteur, liens politique confidentialité et conditions d'utilisation

Bonnes pratiques

  • Utiliser les commentaires pour documenter le code: Sert à expliquer le code et à le rendre plus lisible

  • Utiliser des classes plutôt que des identificateurs Plus flexibles et peuvent être réutilisées plusieurs fois dans une page

  • Organiser le code CSS avec des sections claires Plus lisible

  • Utiliser des noms des classe ecplicites Aide à compredre

  • Tester le code sur différents navigateurs et appareils Tout est dit dans le titre

  • Continuer à aprrendre et à se tenir au courant Le mode du dev web évolue tout le temps

Bonnes pratiques

  • Négliger la compatibilité navigateur Voir vonne pratique

  • Utiliser des sélecteurs inutilement complexe Plus difficile à comprendre et debuguer

  • Oublier de définir une valeur par défaut pour les propriétés Evite des erreurs dans le rendu

  • Négliger l'accessibilité Handicapés et lecteurs d'écrans

  • Ecrire du code CSS redondant Plus difficile à maintenir et debuguer
